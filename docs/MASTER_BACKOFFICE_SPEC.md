# Master Backoffice Spec

Phase AJ status: docs/static master specification only.

Safety boundary: no production DB, no real money, no live provider/payment/bank/SMS/Slip OCR, no migration, no deploy, and no runtime write action is enabled by this document.

## Status Legend

| Status | Meaning |
| --- | --- |
| read-only | Safe display/query surface only. |
| write guarded | Future or existing write surface that must keep auth, permission, reason, audit, and no self-approval where applicable. |
| future live integration | Certification-only until provider/payment/bank/SMS/Slip OCR contracts, sandbox evidence, ledger controls, rollback, and monitoring pass. |

## Cross-Module Requirements

- Backend guards remain the authority; UI hiding is not authorization.
- Read-only permission must exist before each screen is visible.
- Write permission must be separate from read permission.
- Reason is required for wallet, bank approval/reject, blacklist, deposit approval, withdrawal approval/mark paid, promotion, wheel, settings, and provider config writes.
- Audit must include actor, action, target type, target id, before snapshot, after snapshot, reason where required, result, IP mask, user-agent redaction, and createdAt.
- No self-approval for wallet adjustment, withdrawal approval, commission settlement, future expense approval, and any future dual-control action.
- No live action until certification: provider, payment, bank, SMS, Slip OCR, payout, and production DB remain disabled or mock/sandbox only.
