# API Mapping

## Status legend

| Status | Meaning |
| --- | --- |
| mock/static | Screen or behavior is demo-only, mock-only, or static content. |
| backend-ready | Backend route exists, but frontend wiring, staging evidence, or production controls may still be incomplete. |
| API-ready | Route exists and is suitable for safe local/staging frontend integration within the documented boundary. |
| missing | No confirmed dedicated backend route exists yet. |

All rows are constrained by the Safe MVP boundary: no production DB, no real money, no live payout, no live provider/payment/bank/SMS/Slip OCR, no production migration, no production seed, and no deploy approval from this document.

## Cross-cutting API requirements for next integration

| Requirement | Applies to | Current note |
| --- | --- | --- |
| Auth | All member private routes and all admin routes | Keep member JWT and admin JWT separate. |
| Permission | Every admin module | UI menu hiding is advisory only; backend permission guards decide access. |
| Audit log | Admin writes and member money/reward writes | Every future write should define action, actor, target, reason, before/after snapshot, and redaction. |
| Idempotency | Deposit, withdrawal, credit, promotion claim, reward claim, coupon redeem, shop order, game transfer | Existing routes need review before live use; future endpoints should require idempotency keys. |
| Redaction | All responses and logs | No tokens, passwords, secrets, DB URLs, auth headers, raw stack traces, raw user-agent, or unmasked IP values. |
| Safe modes | Provider/payment/bank/SMS/Slip OCR | Only mock, sandbox, or disabled until certification is complete. |
